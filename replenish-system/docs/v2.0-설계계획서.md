## 2. 배치 태그 알고리즘

**파일**: `backend/app/services/wave_builder.py`

### 원칙
- FEFO 절대 원칙 유지 — `matched_bins` 정렬 변경 금지
- groupby 집계 금지 — 행 단위 `batch_tag` / `batch_seq` 컬럼만 부여
- 동일 1순위 보충지번을 `min_group` 이상 공유할 때만 태그 부여

### 구현

```python
def assign_batch_tags(candidates: list[dict], min_group: int = 2) -> list[dict]:
    bin_groups: dict[str, list] = defaultdict(list)
    for c in candidates:
        bins = c.get("matched_bins") or []
        if not bins:
            continue
        primary_bin = bins[0].get("replenish_bin")
        if primary_bin:
            bin_groups[primary_bin].append(c)

    for replen_bin, group in bin_groups.items():
        if len(group) < min_group:
            continue
        sorted_group = sorted(group, key=lambda x: x.get("risk_score", 0), reverse=True)
        for seq, c in enumerate(sorted_group, 1):
            c["batch_tag"] = replen_bin
            c["batch_seq"] = seq
    return candidates
```

### 결과 예시

```
보충지번 15RA0801001을 3개 SKU가 공유

후보 A: 마켓오 그래놀라  batch_tag=15RA0801001, batch_seq=1
후보 B: CJ 햇반          batch_tag=15RA0801001, batch_seq=2
후보 C: 농심 신라면      batch_tag=15RA0801001, batch_seq=3

→ 같은 작업자 배정 → 파렛트 1회 승하강으로 3건 처리
```

## 3. Slack 메시지 v2 (현장 형식)

**파일**: `backend/app/services/slack_service.py`

### 형식

```
*R존 웨이브명*

[📦 15RA0801001]
  1. 15RA1402201  SKU000123  마켓오 그래놀라  15RA0801001
  2. 15RB0501001  SKU000456  CJ 즉석밥        15RA0801001
3. 15NC0312001  SKU001011  파스퇴르 주스    15NC0101001

```

- **지게차(FORKLIFT)**: 배치 태그 묶음 우선 `[📦 헤더]` 표시
- **도보(WALKING)**: 보충지번 순서로 정렬 (L카트 동선 최적화)
- **6개 단위 메시지 분할** (배치 헤더는 카운트 제외)

### 채널 라우팅 — `build_wave_messages_v2()`

```
반환 키 형식: "{channel}_{group}"
  group ∈ {"forklift", "walking", "junior"}

예시:
  R존_forklift  → 지게차 작업자 메시지 (기본)
  R존_walking   → R존 도보 전용 작업자 (별도 메시지)
  R존_junior    → 미숙련 작업자 (후순위 SKU, 별도 메시지)
  P존_walking   → P존 도보 작업자
```

## 4. 선보충 동적 컷오프

**파일**: `backend/app/services/wave_builder.py`

### 공식

```
max_sku = active_workers(FORKLIFT) × prestock_uph × (prestock_minutes / 60)
```

기본값: 3명 × 12 UPH × (100/60) = **60 SKU**

### API

```
GET  /api/v1/waves/cutoff/prestock     컷오프 미리보기
POST /api/v1/waves
  body: {"wave_type": "PRESTOCK"}      max_candidates 자동 산출
```

PRESTOCK 웨이브 생성 시 컷오프 초과분은 risk_score 하위부터 자동 삭제.
