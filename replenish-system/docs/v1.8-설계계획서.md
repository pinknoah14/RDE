**문제**

**해결**  
`layout.tsx` — `<main>` 내부에 `max-w-[1440px]` 센터 래퍼 추가.

```tsx
<main className="flex-1 overflow-y-auto">
  <div className="mx-auto w-full max-w-[1440px]">
    {children}
  </div>
</main>
```

### 4. 지번 마스터 CSV 업로드

#### 4-1. 목적

피킹지번 마스터와 보충/보류지번 마스터 CSV를 시스템에 적재하여:
- 전체 지번 목록을 DB에 보존
- CSV 내 존코드를 자동으로 `zone_config`에 등록 (신규 존코드만)

#### 4-2. 입력 CSV 컬럼

```
센터 창고 | 존 | 작업존 지번 설명 | 작업구역 지번명 상태 |
입고가능 | 피킹가능 | Capa체크 | 가로(mm) | 세로(mm) | 높이(mm) |
CBM | 제품 혼적 | Lot. 혼적 | 생성일자 | 생성시간 | 생성자
```

인코딩: UTF-8-SIG → CP949 순으로 자동 감지.

#### 4-3. 신규 테이블: `bin_master`

```
bin_master
├── bin_id              TEXT  PK   (작업존 지번 설명)
├── center_cd           TEXT
├── zone_prefix         TEXT       (존 컬럼)
├── bin_type            TEXT       PICKING | REPLENISH
├── can_receive         BOOL
├── can_pick            BOOL
├── width_mm            INT?
├── height_mm           INT?
├── depth_mm            INT?
├── cbm                 REAL?
├── allow_mixed_product BOOL
├── allow_mixed_lot     BOOL
├── description         TEXT?
├── status              TEXT?
└── updated_at          DATETIME
```

#### 4-4. 엔드포인트

```
POST /api/v1/upload/bin-master?bin_type=PICKING
POST /api/v1/upload/bin-master?bin_type=REPLENISH
```

**처리 흐름**

```
CSV 수신
  └─ 인코딩 감지 (UTF-8-SIG / CP949)
  └─ polars 파싱 (컬럼명 유연 매핑)
  └─ 행별 처리
       ├─ BinMaster UPSERT (bin_id 기준)
       └─ zone_prefix 수집
  └─ zone_config 자동 등록
       ├─ 기존 존 → zones_existing 카운트
       └─ 신규 존 → INSERT (access_type=FORKLIFT, list_section=MAIN)
  └─ 응답: { bins_upserted, zones_created, zones_existing }
```

#### 4-5. 프론트엔드 변경

**`api.ts`**
```ts
uploadPickingBins: (file) => upload("/upload/bin-master?bin_type=PICKING", file)
uploadReplenishBins: (file) => upload("/upload/bin-master?bin_type=REPLENISH", file)
```

**`upload/page.tsx`** — 섹션 분리

```
운영 데이터 (grid-cols-3)
  ├── 재고현황 CSV
  ├── 출고현황 CSV
  └── 피벗테이블 CSV

지번 마스터 (grid-cols-2)
  ├── 피킹지번 마스터 CSV
  └── 보충/보류지번 마스터 CSV
```

결과창에 `bins_upserted`, `zones_created`, `zones_existing` 표시 추가.
