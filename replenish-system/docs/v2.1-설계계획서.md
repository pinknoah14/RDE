## 2. 배치 그룹 중간 절단 방지

### 배경
v2.0의 `build_wave_message_v2`는 `items_per_msg` 도달 시 단순히 메시지를 끊었기 때문에, 동일한 파렛트 묶음(batch_tag)이 두 메시지로 분할되어 작업자가 헷갈리는 문제가 있었음.

### 구현

**`app/services/slack_service.py`**

```python
def _chunk_preserving_batches(
    line_groups: list[list[str]],
    items_per_msg: int,
) -> list[list[str]]:
    """
    배치 그룹(동일 파렛트 묶음)이 다른 메시지로 찢어지지 않도록 분할.

    규칙:
      - 현재 N개 + 다음 그룹 M개 > items_per_msg 이면 → 다음 메시지로 통째 이동
      - 단일 그룹이 items_per_msg×2 초과 시 예외적 분할 허용
        (파렛트 1개에 12개+ SKU 혼적인 극단적 케이스)
    """
```

`build_wave_message_v2`를 그룹 단위(list[list[str]])로 라인을 구성한 뒤 `_chunk_preserving_batches`에 전달하도록 재작성.

### 검증
- `test_batch_group_not_split_across_messages`: 4 + 3 케이스 → 2번 메시지로 통째 이동 ✅
- `test_batch_group_fits_exactly`: 3 + 3 = 6 → 한 메시지 ✅
- `test_batch_group_oversized_splits`: 14개 단일 그룹 → 예외 분할 허용 ✅
- `test_build_wave_message_v2_no_split_within_batch`: 통합 시나리오 ✅

## 3. weight_unassigned 현장 조정 가이드

### 배경
미할당 가중치(weight_unassigned)는 현장 운영 패턴에 따라 ±10 정도 조정이 필요한 핵심 파라미터. 운영자가 시스템 설정 화면에서 "지금 값을 올려야 하는가?"를 즉시 판단할 수 있도록 가이드를 제공.

### 구현

**`app/core/database.py`** — SYSTEM_CONFIG_SEED:
```python
(
    "weight_unassigned", "15", "INTEGER", "ALGORITHM",
    "미할당 발생 위험도 가중치",
    (
        "기본값: +15\n\n"
        "[현장 조정 가이드]\n"
        "실운영 2~3 사이클 후 아래 기준으로 조정:\n\n"
        "· CRITICAL 추천이 늦다 (미할당 터지고 나서야 추천됨)\n"
        "  → +20 또는 +25로 상향\n\n"
        "· CRITICAL이 너무 많아 관리 불가\n"
        "  → 현재값(15) 유지\n\n"
        "권장 조정 범위: +15 ~ +25\n"
        "최대값 +25 초과 시 다른 가중치와 균형 붕괴 위험"
    ),
),
```

**API**: `GET /api/v1/system-config` 응답에 `description` 필드 노출 (기존 모델/API가 이미 지원).

**Frontend**: `settings/system/page.tsx`에서 multi-line description을 `<details>/<summary>` 토글로 표시:
- 첫 줄만 보이고 클릭 시 전체 가이드 펼침
- `whitespace-pre-line` + `bg-gray-50`로 가독성 확보

### 검증
- `test_weight_unassigned_description_in_api` ✅
- 수동 확인: API 응답에 "현장 조정 가이드" + "+25" 문구 포함 ✅
